如何向OpenStack社区贡献代码

1. 创建账号

创建一个 Launchpad（https://launchpad.net）账号

1.2 登录（https://review.openstack.org），这里会使用Launchpad账号登录。

2. 设置账户信息

参考:http://www.cnblogs.com/openstackteam/p/6108767.html

3. 设置git(先下载代码，后配置git)

3.1 安装git与git-review

3.2 设置git信息

# 与Launchpad账号信息一致

git config --global user.name ``"Firstname Lastname"

git config --global user.email ``"your_email@youremail.com"

git remote add gerrit https://Rambo:1zs84CGBx8gVKQ9xMBQdjLZES7@review.openstack.org/openstack/nova.git

git remote add origin git://git.openstack.org/openstack/nova.git

查看git 源：git remote -v

用户名：Rambo

用户密码：1zs84CGBx8gVKQ9xMBXdjLZES7

git config –list

3.3 下载代码

git clone https://git.openstack.org/openstack/nova

3.4 确保代码是最新的

git checkout master

git pull

3.5 在本地创建并切换到一个新的代码分支（如果是blueprint，分支名是“bp/BP-NAME”，其中的BP-NAME是在launchpad上bp的名称；如果是修复bug，分支则是“bug/BUG-NUMBER”）

git checkout -b bug/xxxxxxx

xxxxxxx为在launchpad里面提交的Bug ID。

3.6 检查git review是否可以提交代码

git review -s

4. 修改代码

5. 提交代码

5.1 添加当前branch的代码修改到本地暂存区域

git add .

5.2 添加提交消息文件

vi .git/message

示例（注意每行最好不要超过70字符）：

Optimize time trigger: fix update trigger property

Optimize time trigger in several aspects. This patch is the third
phase, which fixes a bug that one operation may be triggered at the
same time in both old and new threads when updating the trigger property
two times successively.

Closes-Bug: #xxxxxxx

Implements: blueprint volume-backed-server-rescue

注意：xxxxxxx为在launchpad里面提交的Bug ID

​ Implements为相关blueprint

详细的代码提交信息，参见：https://wiki.openstack.org/wiki/GitCommitMessages
修改完代码后，记得跑通UT测试。

5.3 拷贝commit-msg文件到.git/hooks/目录下。（chmod 777 commit-msg添加权限）

5.4 本地提交

git commit -a -F .git/message

5.5 提交到OpenStack的官方地址

git review

注意：git环境中的账户信息必须与lanchpad账户信息一致，这样才能提交成功，上面是采用密码的形式提交，还有ssh（cat ~/.ssh/id_rsa.pub 将虚机的公钥放到review.openstack.org上）与git两种方式进行提交。

6. review代码

6.1 登录（https://review.openstack.org）查看提交的代码

6.2 根据不同的项目，提交的代码会不同的自动化测试项目，包括单元测试和集成测试。

6.3 如果自动测试没有通过，导致review -1，则需要检查代码是否有问题，点击测试项的名称可以查看相关日志。

6.4 重新修改代码，继续提交

git add -A
git commit –amend （这个里面改message）
git log
git review master (推送到master)

过程中可能遇到的问题：

重复提交过程中本地git日志但并未提交到review.openstack.org

2640a5854978cb88eaf143685ccf843e1818da6d为初次提交的commit号

git diff 2640a5854978cb88eaf143685ccf843e1818da6d
git reset –hard 2640a5854978cb88eaf143685ccf843e1818da6d

6.5 登录（https://review.openstack.org）查看提交的代码，在提交的历史记录里面会显示Uploaded patch set 2

6.6 如果导致自动测试失败的不是自己的代码问题，则可以在回复一个recheck或recheck no bug，将所有测试项重新测试一遍。单个项目的recheck可根据该测试项提示的内容操作。

6.7 如果自动测试项目通过，所有测试项都review+1，就可以让其他开发者进行review，普通review只能+1，核心开发者可以+2，当提交的代码收到2个核心开发者+2后，就能Merge到目标分支.
